then tell Claude to apply the approved text to `data/measures.json`.

(e.g., P1.F3.A4) for future deep linking to pillar page dashboards. The `linkText`
("See Pillar X") uses the **action's** pillar, not the measure's pillar — so
cross-pillar cards will display in their own pillar color.

## Selection Criteria Applied

1. **One-hop rule:** Every cross-pillar link is at most one logical step from the student outcome. Example: "Instructional Coaching" (P2) → better daily instruction → higher ACT scores. Pass.
2. **Closer-to-student rule:** All selected actions touch classrooms, schools, or families directly. No agency-internal actions (except bench P5 entry).
3. **Specificity test:** Every description names a concrete mechanism. No vague "this supports student success" language.
4. **No forced links:** P5 excluded from primaries because no action met the one-hop test credibly.
5. **No over-use:** Maximum 2 primary uses per action (see table above).
